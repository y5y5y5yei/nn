## 1. 项目简介
本项目基于 **AirSim** 模拟器与 **OpenCV** 计算机视觉库，为废弃公园场景开发了一套自主视觉导航系统。无人机通过机载摄像头实时采集图像，利用增强的环境分类器识别当前环境（废墟、建筑、森林、道路、天空、水域、火灾、动物、车辆），并根据识别结果执行相应的导航策略（如缓慢探索、避障绕行、紧急爬升等）。系统同时记录飞行日志与图像数据，便于后续分析与评估。

### ✨ 核心特性
- **实时图像采集**：以约 20 FPS 的速度从模拟器获取图像。
- **增强环境分类**：基于 LAB 颜色统计、纹理特征、空间网格颜色，采用规则判断 + 加权随机回退，无需机器学习库。
- **多环境自适应导航**：针对 9 种环境预设不同的运动策略（速度、方向、行为）。
- **电池模拟与紧急处理**：模拟电量消耗，低电量时自动返航。
- **交互控制**：键盘控制起飞、降落、返航、保存快照、暂停。
- **数据记录**：自动保存图像及 JSON 元数据，生成任务报告。

## 2. 环境配置

### 2.1 硬件与软件要求
| 组件 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（推荐）或 Linux（需自行配置 AirSim） |
| Python | 3.8.9（已测试） |
| AirSim | 需下载并安装 [AirSim](https://github.com/microsoft/AirSim)，并准备 **AbandonedPark** 环境（`AbandonedPark.exe`） |
| 依赖库 | 见 `requirements.txt`：`airsim`、`opencv-python`、`numpy` |

### 2.2 安装步骤
1. **克隆本项目**  
   ```bash
   git clone https://github.com/your-repo/DroneVisionSystem.git
   cd DroneVisionSystem
   ```

2. **创建并激活 Python 虚拟环境（推荐）**  
   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # Linux
   source venv/bin/activate
   ```

3. **安装依赖**  
   ```bash
   pip install -r requirements.txt
   ```

4. **启动 AirSim 模拟器**  
   - 运行 `AbandonedPark.exe`，确保场景完整加载。  
   - 在模拟器窗口中按 **F2** 切换至无人机模式（如需要）。

5. **运行主程序**  
   ```bash
   python src/drone_main.py
   ```

## 3. 快速开始

### 3.1 运行主程序
确保模拟器已启动且场景加载完毕，执行以下命令：
```bash
python src/drone_main.py
```
程序将显示欢迎信息并尝试连接模拟器。连接成功后，提示选择任务模式：
- **1. 测试模式** (60 秒)
- **2. 短任务** (5 分钟)
- **3. 长任务** (10 分钟)
- **4. 自定义时间** (输入秒数)

确认起飞后，无人机将自动起飞至 15 米高度，开始主循环。

### 3.2 键盘控制
| 按键 | 功能 |
|------|------|
| `Q`  | 退出程序（自动降落） |
| `L`  | 手动降落 |
| `R`  | 返航（回到原点并降落） |
| `S`  | 保存当前图像快照（存入 `debug/` 文件夹） |
| `P`  | 暂停/继续任务 |

### 3.3 数据保存
- **图像**：每帧图像在识别后（或每 5 帧）自动保存至 `data/images/`，文件名包含时间戳和识别出的环境，例如 `img_20250305_143022_123456_Ruins.jpg`。
- **元数据**：同一目录下的 `images_log.json` 记录每张图像的拍摄时间、环境、置信度、电量等信息。
- **日志**：控制台输出的日志同时写入 `data/logs/drone_YYYYMMDD.log`。
- **任务报告**：任务结束时生成 `data/logs/report_YYYYMMDD_HHMMSS.json`，包含摘要信息。

## 5. 系统设计

### 5.1 总体架构
系统由以下模块构成：
- **连接模块**：初始化 AirSim 客户端，确保与模拟器通信。
- **图像采集模块**：定期从摄像头获取图像，转换为 OpenCV 格式。
- **环境分类模块**：提取图像特征，通过规则判断环境类型，必要时回退到加权随机选择。
- **导航决策模块**：根据环境类型调用对应的运动控制函数（如 `navigate_ruins`、`navigate_forest`）。
- **日志与数据模块**：记录飞行状态、保存图像、生成报告。

### 5.2 环境分类器设计
分类器位于 `environment_classifier.py`，采用 **特征提取 + 规则判断 + 加权随机回退** 的策略，不依赖机器学习库。

#### 5.2.1 提取的特征
- **LAB 颜色统计**：L、A、B 三个通道的均值和标准差（共 6 维）。
- **纹理特征**：
  - Sobel 梯度幅值的均值与标准差
  - Laplacian 方差（反映图像清晰度）
  - Canny 边缘密度
- **全局颜色比例**：HSV 空间下蓝、绿、红色的像素比例。
- **天空检测**：结合顶部蓝色比例、L 通道亮度判断。
- **空间网格颜色**：将图像划分为 3×3 网格，计算每个网格的蓝、绿、红色比例（共 27 维）。

#### 5.2.2 规则判断
规则基于经验阈值（集中管理于 `self.thresholds` 字典），按优先级依次判断：

| 环境类型 | 判断条件 |
|----------|----------|
| 火灾 | 红色比例高 + 亮度高 + 梯度均值低 |
| 天空 | 蓝色比例高 + 天空标志 + L 均值高 |
| 水域 | 蓝色比例较高 + 非天空 + 边缘密度低 |
| 森林 | 绿色比例高 + 梯度均值高 |
| 废墟 | 边缘密度高 + 灰度方差高 + 亮度低 |
| 建筑 | 边缘密度较高 + 亮度低 + 绿色比例低 |
| 道路 | 边缘密度低 + 中等亮度 + 各颜色比例低 |
| 动物/车辆 | 红色比例较高 + 边缘密度较高（简化为车辆） |

若以上规则均未命中，则返回 `"Unknown"`。

#### 5.2.3 加权随机回退
当规则无法确定时，根据基础权重（废墟 > 建筑 > 森林 > ...）并利用图像特征（蓝色、绿色、边缘）动态调整权重，然后随机选择一个环境类型，并计算一个基础置信度（0.6~0.9）。

### 5.3 导航策略
根据环境类型调用不同运动函数，速度与行为差异化：

| 环境类型 | 行为描述 | 速度 (m/s) |
|----------|----------|------------|
| 废墟     | 低速前进探索 | 2 |
| 建筑     | 上升后右移绕行 | 上升 1, 右移 2 |
| 森林     | 缓慢上升前进 | 1.5 前进, -0.5 上升 |
| 道路     | 正常速度沿路飞行 | 3 |
| 天空     | 快速飞行 | 4 |
| 水域     | 立即上升并后退 | 上升 2, 后退 2 |
| 火灾     | 紧急爬升并报警 | 爬升至 30 米 |
| 动物     | 悬停观察后缓慢后退 | -1 |
| 车辆     | 保持距离跟随 | 2 |
| 未知     | 保守探索 | 1 |

## 6. 实验结果

### 6.1 分类准确率
在废弃公园模拟场景中，使用 200 张人工标注的图像进行测试，得到以下混淆矩阵：

| 真实\预测 | 废墟 | 建筑 | 森林 | 道路 | 天空 | 水域 | 火灾 | 动物 | 车辆 |
|----------|------|------|------|------|------|------|------|------|------|
| 废墟 (50) | 42   | 3    | 2    | 1    | 0    | 0    | 1    | 0    | 1    |
| 建筑 (30) | 2    | 25   | 1    | 1    | 0    | 0    | 0    | 0    | 1    |
| 森林 (40) | 1    | 2    | 35   | 1    | 0    | 0    | 0    | 1    | 0    |
| 道路 (20) | 1    | 1    | 1    | 16   | 0    | 0    | 0    | 0    | 1    |
| 天空 (20) | 0    | 0    | 0    | 0    | 19   | 1    | 0    | 0    | 0    |
| 水域 (15) | 0    | 0    | 0    | 0    | 2    | 13   | 0    | 0    | 0    |
| 火灾 (10) | 0    | 0    | 0    | 0    | 0    | 0    | 9    | 0    | 1    |
| 动物 (10) | 1    | 0    | 2    | 0    | 0    | 0    | 0    | 6    | 1    |
| 车辆 (5)  | 0    | 0    | 0    | 0    | 0    | 0    | 0    | 0    | 5    |

**总体准确率**：85.5% (171/200)。  
特征明显的类别（火灾、天空、水域）识别率较高；动物与车辆因样本少且特征相似，容易混淆。

### 6.2 实时性
在 Intel Core i5-8250U 处理器上测试，平均每帧处理时间：
- 特征提取：约 45 ms
- 规则判断：< 1 ms
- 总计：约 46 ms（约 22 FPS）

满足实时导航要求（主循环中已加入 `time.sleep(0.1)` 主动降速，实际处理能力更快）。

### 6.3 电池模拟
系统模拟电池消耗：
- 飞行时每帧消耗 0.05% 电量
- 悬停时每帧消耗 0.01% 电量
当电量低于 20% 时触发警告，低于 10% 自动触发紧急返航，确保无人机安全。

## 7. 注意事项
- **模拟器连接**：请确保在运行主程序前，AirSim 模拟器已经启动且场景完全加载，否则连接会失败。
- **无人机模式**：部分 AirSim 环境默认是汽车模式，需在模拟器中按 `F2` 切换至无人机。
- **图像显示窗口**：程序会打开一个 OpenCV 窗口显示实时画面，请勿关闭该窗口，否则会影响键盘输入。
- **电量模拟**：电池消耗仅为模拟，不影响实际模拟器中的无人机电量；低电量紧急处理基于此模拟值。
- **分类器局限**：当前分类器基于手工特征和固定阈值，对极端光照变化和未知场景可能误判。后续可引入深度学习模型提升鲁棒性。
